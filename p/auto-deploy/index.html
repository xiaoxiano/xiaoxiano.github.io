docker自动化部署

2021-11-15

actions部署前后端分离项目

参考文章

本文使用github actions + docker compose部署到github container registry（这适用于中小型项目） 附：容器注册平台对比

  1. 准备好你的dockerfile，如无docker和docker-compose基础请先看文档和前文。

  2. 使用github actions

    “点击展开示例actions”

      1on:
  2  push:
  3    branches:
  4      - master
  5      
  6  workflow_dispatch:
  7
  8env:
  9  REGISTRY: ghcr.io
 10  IMAGE_NAME: ${{ github.repository }}
 11
 12jobs:
 13          
 14  build-and-push-image:
 15    runs-on: ubuntu-latest
 16    permissions:
 17      contents: read
 18      packages: write
 19    strategy:
 20      matrix:
 21        node: [ '14' ]    
 22
 23    steps:
 24      - name: Checkout repository
 25        uses: actions/checkout@v2
 26
 27      - name: Setup node
 28        uses: actions/setup-node@v2
 29        with:
 30          node-version: ${{ matrix.node }}
 31          
 32      - name: npm install
 33        run: |
 34          cd web                       #进入前端dockerfile目录
 35          npm install       
 36              
 37      - name: Set up QEMU
 38        uses: docker/setup-qemu-action@v1
 39      
 40      - uses: docker/setup-buildx-action@v1
 41        id: builder1
 42      
 43      - uses: docker/setup-buildx-action@v1
 44        id: builder2
 45        
 46      - name: Log in to the Container registry
 47        uses: docker/login-action@v1
 48        with:
 49          registry: ${{ env.REGISTRY }}
 50          username: ${{ github.actor }}
 51          password: ${{ secrets.GITHUB_TOKEN }}
 52          
 53      # 用于消毒标签，tag是此动作的重要功能之一，本示例没使用，请见文档    
 54      - name: Extract metadata (tags, labels) for Docker
 55        id: meta1
 56        uses: docker/metadata-action@v3
 57        with:
 58          images: ghcr.io/username/server      # images地址
 59      - name: Extract metadata (tags, labels) for Docker
 60        id: meta2
 61        uses: docker/metadata-action@v3
 62        with:
 63          images: ghcr.io/username/web         # images地址
 64
 65
 66      - name: Build and push Docker image 1
 67        uses: docker/build-push-action@v2
 68        with:
 69          builder: ${{ steps.builder1.outputs.name }}        
 70          context: ./server                # 服务端dockerfile目录
 71          file: ./server/Dockerfile        # 服务端dockerfile
 72          push: true
 73          tags: ${{ steps.meta1.outputs.tags }}
 74          labels: ${{ steps.meta1.outputs.labels }}
 75          cache-from: type=gha
 76          cache-to: type=gha,mode=max
 77
 78      - name: Build and push Docker image 2
 79        uses: docker/build-push-action@v2
 80        with:
 81          builder: ${{ steps.builder2.outputs.name }}        
 82          context: ./web                  # 前端dockerfile目录
 83          file: ./web/Dockerfile          # 前端dockerfile
 84          push: true
 85          tags: ${{ steps.meta2.outputs.tags }}
 86          labels: ${{ steps.meta2.outputs.labels }}
 87          cache-from: type=gha
 88          cache-to: type=gha,mode=max
 89  deploy:
 90    runs-on: ubuntu-latest
 91    needs: build-and-push-image
 92    steps:
 93              
 94      - uses: appleboy/ssh-action@master
 95        with:
 96          host: ${{ secrets.host }}          #仓库设置中添加Secrets：HOST
 97          username: root
 98          password: ${{ secrets.PASSWORD }}  #仓库设置中添加Secrets：PASSWORD
 99          port: 22
100          script: |         
101            echo $CR_PAT | docker login ghcr.io -u USERNAME --password-stdin     # 用户名
102            docker-compose -f docker-compose-nginx.yaml -f docker-compose-app.yaml pull   # docker-compose名
103            docker-compose -f docker-compose-nginx.yaml -f docker-compose-app.yaml up -d   # docker-compose名
104
105# 此actions许多参数由github自动提供，无需添加，需修改的均已注释

  3. 在服务器上完成向 Container registry 验证

具体操作：打开/etc/profile

1export CR_PAT=YOUR_TOKEN
2source /etc/profile
3echo $CR_PAT | docker login ghcr.io -u USERNAME --password-stdin
  1. 新建docker-compose-app.yaml和代理容器docker-compose-nginx.yaml,示例。如有不懂，请多看几遍acme-companion和上述示例文档。
